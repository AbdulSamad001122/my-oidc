OIDC Provider Documentation

Welcome to the MyOIDC developer documentation. This guide explains how to integrate your applications with our authentication server using the standard OpenID Connect (OIDC) Authorization Code flow.

1 Register Your Application

Before your application can authenticate users, it must be registered to obtain a Client ID and Client Secret.

  1. Go to the Client Registration Portal.
  2. Create an account or log in as a developer.
  3. Click "Register New Client" and provide your application's Name and Redirect URIs.
  4. Save your Client ID and Client Secret securely. You will need these for the OAuth flow.

2 Discovery & Configuration

Our OIDC provider supports standard discovery endpoints, allowing OIDC client libraries to automatically configure themselves.

GET /.well-known/openid-configuration 
// Returns provider metadata:
{
  "issuer": "http://localhost:8000",
  "authorization_endpoint": "http://localhost:8000/o/authorize",
  "userinfo_endpoint": "http://localhost:8000/o/userinfo",
  "jwks_uri": "http://localhost:8000/.well-known/jwks.json"
}

3 Authorization Code Flow

A. Redirect to Authorize Endpoint

Redirect the user's browser to the authorization endpoint to prompt them to log in and grant consent.

GET /o/authorize 
GET /o/authorize?client_id=YOUR_CLIENT_ID
  &redirect_uri=YOUR_REDIRECT_URI
  &response_type=code
  &state=RANDOM_STRING

After successful login and consent, the server will redirect the user back to your redirect_uri with a code parameter.

B. Exchange Code for Tokens

From your backend server, make a POST request to exchange the authorization code for an ID Token and Access Token.

POST /o/token 
POST /o/token
Content-Type: application/json

{
  "grant_type": "authorization_code",
  "code": "AUTH_CODE_FROM_URL",
  "client_id": "YOUR_CLIENT_ID",
  "client_secret": "YOUR_CLIENT_SECRET",
  "redirect_uri": "YOUR_REDIRECT_URI"
}

Response

{
  "access_token": "random_access_string",
  "id_token": "eyJhbGci...", // JWT containing user identity
  "token_type": "Bearer",
  "expires_in": 3600
}

C. Fetch User Info (Optional)

Use the obtained access token to retrieve user profile information from the userinfo endpoint.

GET /o/userinfo 
GET /o/userinfo
Authorization: Bearer YOUR_ACCESS_TOKEN

4 Testing with Postman

You can easily test the OAuth 2.0 flow using Postman's built-in authorization tab:

  • Select OAuth 2.0 in the Authorization tab.
  • Set Grant Type to Authorization Code.
  • Auth URL: http://localhost:8000/o/authorize
  • Access Token URL: http://localhost:8000/o/token
  • Provide your Client ID and Client Secret.
  • Click Get New Access Token to start the browser flow!
